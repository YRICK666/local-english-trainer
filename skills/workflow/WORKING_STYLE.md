# Working Style

本文件记录用户与 ChatGPT / Codex 协作维护 local-english-trainer 时的工作习惯。

## 1. 沟通方式

- 默认使用中文沟通；
- 说明要具体，不要空泛总结；
- 能直接给命令时，优先给可复制运行的 PowerShell 脚本；
- 创建多个文件或目录时，直接给一整段脚本，不要拆成多个单独步骤；
- 如果需求不明确，先用一句话复述卡点再继续，不要直接猜。
- 用户当前明确指令优先；如果用户要求不运行 Git、不跑测试、只评估不改文件，就按用户当前指令执行，并说明哪些默认检查未执行。

## 2. 项目偏好

项目定位：

    英语阅读训练 + 本地单词库搭建

前端风格：

    好看、简约、清爽，适合学习工具，不复杂花哨。

命名偏好：

- 不使用 v1、v2 这类项目命名表达；
- 文档标题尽量直观；
- 文件名使用稳定、可长期维护的名称。

## 3. 开发边界

当前阶段优先完成最小学习闭环。

暂不提前扩展：

- OCR；
- AI 自动解析；
- 云服务；
- 多用户系统；
- 登录注册；
- 复杂复习算法；
- 过度工程化的插件系统。

## 4. Git 操作习惯

必须保守操作 Git。

禁止随意使用：

    git add -A
    git add .
    git clean -fdX
    git reset --hard
    git push --force

提交前必须检查：

    git status
    git ls-files | Select-String "node_modules|__pycache__|sqlite3|pytest_runtime_tmp|frontend/dist|frontend/build-output|\.log"

第二条命令必须没有输出。

## 5. 阶段汇报习惯

每个阶段完成后，要输出具体阶段报告，包括：

- 本阶段目标；
- 实际完成内容；
- 修改文件；
- 接口变化；
- 前端变化；
- 测试命令和结果；
- 已知问题；
- 下一步建议。

不要只写已完成。

验证说明按任务类型写：

- 后端任务：说明 pytest 或相关后端测试结果；
- 前端任务：说明 npm run build 结果；
- 纯文档任务：可以写“本阶段未运行，因为未修改代码”；
- 纯评估任务：默认不修改文件、不运行测试、不运行 Git 命令，除非用户明确要求。
