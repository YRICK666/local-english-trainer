# Agent Workflow

本文件补充 `AGENTS.md` 的长期执行规则。已写入此处的规则默认适用于后续任务；提示词只需补充本轮目标、允许文件、禁止文件和特殊风险。

## 开始、继续与停止

- 开始时只做一次必要检查：项目根目录、`AGENTS.md`、任务相关文件、Git 状态和真实数据边界。
- 已知、已声明且属于当前阶段的未提交中间修改是工作基线，不是仓库污染；保留它们并在其基础上继续。staged 内容仍需单独确认。
- 最终验收条件不能自动升级为开始条件。例如“工作区干净”只在任务明确要求时才是开始门槛；已声明的阶段中间状态不应阻塞继续实施。
- 每轮只完成一个可验证的小闭环；开始必要检查一次，结束再做完整范围、测试和 Git 验证。

## 必须停止的情形

仅在以下情形停止并请求方向：

1. staged 内容来源不明；
2. 出现任务范围外的未知修改；
3. 后续动作可能访问、覆盖、迁移或破坏真实数据；
4. 需要高风险 Git、进程或系统操作，且用户没有明确授权；
5. 无法确定现有代码语义，继续可能造成破坏。

## 应自动修复并继续的情形

下列是普通工程问题，不是停止条件：补丁或文本锚点失败、路径/命令问题、首次编译失败、单元测试失败、unused warning、非必要工具缺失、格式或换行差异。

处理顺序：完整读取实际文件与错误 → 判断原因 → 换用符号定位、受控行级编辑或受控整文件重写 → 修复 → 重新验证。不得因一次失败自动放弃；也不得用 `#[allow(dead_code)]`、跳过测试或扩大权限掩盖问题。

完整理解目标文件后，允许基于 Rust/TypeScript/Python 符号和实际结构编辑；当局部补丁不可靠时，允许保留现有逻辑的受控整文件重写。每次写入后立即读取并执行 `git diff --check`。

warning 只有在当前小阶段明确要求消除、或它表明实际错误时才是阻塞项；否则记录来源并继续完成当前闭环。

## 长期安全边界

- 未经明确授权，不执行 Git 写入（`add`、`commit`、`push`、`reset`、`restore`、`clean` 等）。
- 不访问、读取、复制、迁移或修改真实 SQLite；测试仅可使用任务明确创建和归属可确认的临时根。
- 不 broad-kill 或按进程名终止用户进程；只操作本轮保存的精确 child/PID。
- 不修改永久 PATH、代理、系统安全设置、ACL 或工具链，除非用户明确授权。
